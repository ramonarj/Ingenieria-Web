Firefighter Space

Objetivo

Creación de una aplicación web que ofrezca a un cuerpo de bomberos concreto gestionar información de forma rápida teniéndola toda abordada en una misma web accesible solo con una cuenta de bombero.

Soporta los siguientes roles:

  • El administrador puede dar de alta bomberos, añadir nuevos tipos de herramientas y aceptar o denegar cambios de turno.
  • Un usuario tiene acceso a su información personal. También puede ver información de cómo están gestionadas algunos aspectos del parque. Además tiene la posibilidad de ver sus turnos y cambiarlos si es necesario. Por último, puede acceder a un apartado de chat donde tiene la posibilidad de comunicarse con otros usuarios.

Vistas

login Vista del login. Permite al usuario hacer login y entrar directamente como user o admin.
inicio Vista del menú de inicio. Permite elegir desde un principio entre las cuatro funciones que puede hacer un usuario normal. En el caso de que sea admin, tendrá una función más, pero esta no se verá representada en este menú, sino que se verá en la toolbar.
chat Vista del apartado de chat. Esta funcionalidad permite al usuario poder comunicarse con los demás usuarios de la aplicación.
chat Vista del apartado de chat. Esta funcionalidad permite al usuario poder comunicarse con los demás usuarios de la aplicación.
equipo Vista del equipo. Aquí, el usuario podrá ver todas aquellas herramientas asignadas a cada bombero del parque, es decir, puede ver quién tiene las linternas, los hachas...
horario Vista del calendario. En esta vista podemos ver un calendario donde están señalados aquellos días en los que el usuario tiene que trabajar. Además hay un apartado que le da la posibilidad de pedir al administrador el cambio de uno de sus turnos por otro de alguno de sus compañeros.
perfil Vista del perfil de usuario. Aquí el usuario podrá ver sus datos personales y cambiar alguno de ellos si es necesario.
admin Vista del admin. Este apartado de administración solo está disponible para los usuarios que sean administradores. La única forma de acceder a esta vista es desde la toolba como se ha comentado anteriormente. Las siguientes vistas que se van a explicar solo están disponibles para el administrador y la forma de acceder a ellas es a través de esta vista.
addUser

Añadir usuario

Vista para añadir usuario. Desde aquí el administrador puede añadir nuevos usuarios a la base de datos, para ello tiene que completar unos parámetros requeridos.
delUser

Eliminar usuario

Vista para eliminar usuario. Al contrario que en la vista anterior, ahora el admistrador puede eliminar a un usuario.
addTool

Añadir herramienta

Vista de añadir herramienta. Desde aquí el administrador puede añadir a la base de datos un tipo más de herramienta.
assignTool

Asignar herramienta

Vista para asignar herramienta. El administrador cuenta con la posibilidad de asignar alguno de los tipos de herramientas a los usuarios registrados.
manageChanges

Peticiones de cambio

Vista para manejar cambios. A esta vista llegan las peticiones de cambio de turno propuestas por los usuarios. El administrador tiene la posibilidad de aceptar o denegar esos cambios.
Por último, comentar la funcionalidad de la toolbar ya mencionada. A través de ella se puede ir a cualquier parte de la aplicación que tienen en común todos los usuarios, es decir, el calendario, el equipo, el chat y el perfil. Podemos ver alguna excepción en el caso de que el usuario sea un administrador, que entonces le aparece un opción para acceder al apartado de administración.

Mejoras

Ahora vamos a hablar de las mejoras post-examen.
login Se han eliminado las opciones "debug" con las que se accedía rápidamente a la aplicación. Para poder probar el funcionamiento de la aplicación entrar con estos usuarios y constraseñas:

    Usuario

    usuario: ale contraseña: ale

    Administrador

    usuario: ramon contraseña: ramon
inicio Si el administrador inicia sesión, le aparecerá directamente la opción para poder acceder al apartado de administración
equipo Se ha mejorado el css y se ha incluido la opción para pedir el cambio de herramienta con alguno de los usuarios, sigue el mismo sistema que la petición del cambio de turno
admin Mejorado el css y se le han añadido más opciones al administrador: las gestión de vehículos, eliminar herramientas, e intercambiar herramientas.
manageChanges Mejorado el css. Ahora también puede gestionar las peticiones de cambios de herramientas.
addUser Mejorado el css.
addAuto

Añadir vehículo

Ahora se pueden añadir tipos de vehículos a la base de datos.
delAuto

Eliminar vehículo

Puedes eliminar el vehículo que quieras. Al inicio no hay vehículos, por lo tanto, no aparecerá nada hasta que se añada alguno.
assignDriver

Asignar conductor

Asigna conductor a una sola persona sin especificar el tipo de vehículo que conducirá.
delTool

Eliminar herramienta

El administrador puede borrar alguna de las herramientas que hay en la base de datos.
switchTool

Intercambiar herramientas

Se pueden intercambiar las herramientas de los usuarios elegidos por el administrador, es decir, los usuarios intercambiarán sus herramientas a voluntad del administrador.
